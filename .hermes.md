# SantriOnline Project Context

## Project README Summary
1|# SantriOnline TPQ
2|
3|SantriOnline adalah aplikasi manajemen TPQ berbasis SvelteKit untuk mengelola santri, hafalan, setoran, halaqoh, sertifikat, dan konten pendukung. Repo ini juga memuat lapisan monetisasi seperti shortlink analytics, buku digital, coin wallet, DRM, digital store, CMS, dan Kitab AI/RAG. Arah pengembangan aktif tetap berfokus pada **TPQ-only mode**, sementara surface legacy non-TPQ dipetakan sebagai alias atau modul lama.
4|
5|## Stack
6|
7|- Svelte 5 + SvelteKit
8|- TypeScript + Vite
9|- Tailwind CSS + DaisyUI
10|- Cloudflare Pages/Workers
11|- Cloudflare D1
12|- Cloudflare R2
13|- Lucia Auth + Arctic
14|- pdf-lib untuk sertifikat PDF
15|- Wrangler untuk deploy dan resource Cloudflare
16|
17|Folder `src-tauri/` tersedia untuk companion desktop app, tetapi aplikasi web tetap source of truth utama.
18|

73|## Catatan Arsitektur
74|
75|- Semua data tenant harus di-scope dengan `org_id` atau `institution_id`.
76|- Route kanonis workflow TPQ adalah `/tpq/akademik/*`.
77|- Route lama seperti `/akademik`, `/dashboard/setoran-hari-ini`, dan `/dashboard/review-setoran` hanya server-only redirect role-aware.
78|- Sertifikat disimpan di R2, metadata-nya ada di D1.
79|- Endpoint utilitas seperti `/api/admin/migrate` dan `/api/seed-admin` hanya untuk maintenance: superadmin-only, secret wajib di production, dan aktivitasnya dicatat di `system_logs`.
80|- Endpoint AI yang mahal harus memakai quota/rate limit dan tidak boleh menjadi surface publik bebas.
81|- Jika menambah fitur baru, prioritaskan konteks TPQ dan hindari memperluas modul legacy non-TPQ.

141|## Role System (Updated 2026-06-04)
142|
143|SantriOnline menggunakan **simplified role-based access control (RBAC)** dengan struktur hierarki:
144|
145|### Platform Roles (Global)
146|- `SUPER_ADMIN` - Akses penuh ke seluruh sistem
147|- `support` - Tim support platform
148|- `auditor` - Audit dan monitoring sistem
149|
150|### Organization Roles (Universal)
151|
152|**Role Levels:**
153|- Level 3 (Admin): `admin`, `kepala`, `bendahara`
154|- Level 2 (Manager): `pembimbing`, `operator`, `sekretaris`, `humas`, `kurikulum`, `pembina`
155|- Level 1 (Staff): `pengajar`, `takmir`, `imam`, `khotib`, `muadzin`
156|- Level 0 (Member): `santri`, `wali`, `alumni`, `jamaah`
157|
158|**Simplified Roles:**
159|- `admin` - Administrator organisasi (full access)
160|- `kepala` - Kepala/Pimpinan (merge: kepala_tpq, kepala_tahfidz, pengasuh, ketua_takmir)
161|- `pengajar` - Pengajar (merge: ustadz, ustadzah - gender di user profile)
162|- `pembimbing` - Pembimbing (merge: musyrif, koordinator, wali_kelas)
163|- `operator` - Operator data
164|- `bendahara` - Pengelola keuangan
165|- `sekretaris` - Administrasi dan dokumentasi
166|- `humas` - Public relations dan media sosial
167|- `kurikulum` - Pengembangan program pembelajaran
168|- `pembina` - Pembinaan karakter dan kedisiplinan
169|- `santri` - Siswa/Santri
170|- `wali` - Wali santri/orang tua
171|- `alumni` - Alumni (pondok, rumah-tahfidz)
172|- `jamaah` - Jamaah (masjid, musholla)
173|- `takmir` - Takmir (masjid, musholla)
174|- `imam` - Imam sholat (masjid, musholla)
175|- `khotib` - Khotib Jumat (masjid, musholla)
176|- `muadzin` - Muadzin (masjid, musholla)
177|
178|### Permission System
179|
180|**Wildcard Support:**
181|```typescript
182|'hafalan.*'  // semua akses hafalan
183|'finance.*'  // semua akses keuangan
184|'org.*'      // semua akses organisasi
185|```
186|
187|**Scope-based Permissions:**
188|```typescript
189|'student.read.own'   // hanya santri sendiri
190|'student.read.class' // satu kelas
191|'student.read.all'   // semua santri
192|```
193|
194|**Multi-role Support:**
195|- User bisa memiliki `secondary_roles` (JSON array)
196|- Temporary role dengan `role_expires_at`
197|- Role delegation untuk temporary permissions
198|
199|### Legacy Role Mapping
200|
201|Migration otomatis dari role lama ke role baru:
202|- `ustadz`, `ustadzah` → `pengajar`
203|- `kepala_tpq`, `kepala_tahfidz`, `pengasuh`, `ketua_takmir` → `kepala`
204|- `koordinator`, `wali_kelas`, `musyrif` → `pembimbing`
205|
206|**Helper Functions:**
207|```typescript
208|import { normalizeRole, isRole, isTeachingRole } from '$lib/utils/role-helpers';
209|
210|// Normalize legacy role
211|const role = normalizeRole('ustadz'); // returns 'pengajar'
212|
213|// Check role (handles legacy)
214|if (isRole(user.role, 'ustadz')) { } // works with both 'ustadz' and 'pengajar'
215|
216|// Check teaching role
217|if (isTeachingRole(user.role)) { } // true for pengajar, ustadz, ustadzah
218|```
219|
220|### Role Assignment Rules
221|
222|1. **Principle of Least Privilege** - Berikan permission minimal yang dibutuhkan
223|2. **Separation of Duties** - Hindari conflict of interest (e.g., bendahara tidak boleh jadi admin)
224|3. **Regular Review** - Audit role assignment setiap 6 bulan
225|4. **Time-bound Roles** - Gunakan `role_expires_at` untuk temporary assignment
226|
227|### Audit & Compliance
228|
229|Semua perubahan role dan penggunaan permission dicatat di:
230|- `user_role_history` - History perubahan role
231|- `permission_usage_logs` - Log penggunaan permission
232|- `role_delegations` - Delegasi permission temporary
